# Security Policy

## Scope

This repository is primarily a JSON dataset (`wmn-data.json`) along with small supporting scripts. Security reports should cover things like:

- Malicious or unsafe code in the scripts in this repo (e.g., something that could lead to remote code execution, SSRF, or similar when the scripts or data are used as intended)
- Vulnerabilities in how the project's tooling processes the data file

If you've found a **site that should be added, removed, or has incorrect detection data** (wrong `e_code`/`e_string`, false positives, etc.), that's not a security issue -- please [open a regular issue](https://github.com/WebBreacher/WhatsMyName/issues) or submit a pull request instead.

## Reporting a Vulnerability

To report a vulnerability, send an email to `micah` `@` `myosint.training` with details of what you found, how we can reproduce it, and any images or videos.

We'll do our best to acknowledge your report within **1 week**.

## Safe Harbor

We support good-faith security research. If you make a good-faith effort to follow this policy, including reporting issues responsibly and not accessing or modifying data beyond what's needed to demonstrate the issue, we won't pursue legal action against you for your research.

Please give us a reasonable amount of time to investigate and address the issue before disclosing it publicly.
